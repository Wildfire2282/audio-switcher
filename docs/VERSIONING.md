# 版本号更新规则

本项目遵循 [语义化版本 2.0.0](https://semver.org/lang/zh-CN/) 规范，版本号格式为 `MAJOR.MINOR.PATCH`。

## 版本号递增规则

| 修改类型 | 递增位 | 示例 |
|---|---|---|
| **破坏性变更** — 移除/重命名公共 API、改变默认行为、不兼容的架构调整 | **MAJOR** | `2.0.5` → `3.0.0` |
| **新增功能** — 向后兼容的新特性（如新增菜单项、新交互方式） | **MINOR** | `2.0.5` → `2.1.0` |
| **Bug 修复 / 小改进** — 向后兼容的问题修复、性能优化、代码清理、UI 微调 | **PATCH** | `2.0.5` → `2.0.6` |

## 当前版本

`2.1.0`

## 如何更新版本号

修改 `AudioSwitcher.csproj` 中的 `<Version>` 属性：

```xml
<Version>2.1.0</Version>
```

菜单中"关于"项会自动从程序集读取并显示 `v2.1.0`。

## 判断指南

- **PATCH**：修复崩溃、修复逻辑错误、清理无用代码、调整 UI 布局、改进日志/提示文本
- **MINOR**：新增菜单项、新增设置选项、新增交互方式、新增本地化语言
- **MAJOR**：重写核心架构、移除现有功能、改变用户可见的默认行为、改变配置文件格式

## 示例

| 改动 | 类型 | 版本变化 |
|---|---|---|
| 修复开机自启偶尔失效 | PATCH | `2.0.5` → `2.0.6` |
| 新增托盘图标双击切换设备 | MINOR | `2.0.5` → `2.1.0` |
| 移除"刷新设备列表"菜单项 | MAJOR | `2.0.5` → `3.0.0` |
